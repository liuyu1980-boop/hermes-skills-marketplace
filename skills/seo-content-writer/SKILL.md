# SEO Content Writer

Write articles that actually rank — data-driven, competitor-aware, not generic "10 tips" listicles.

## Trigger

Load when the user gives a keyword and wants an SEO article, says "write a blog post about X that ranks," or wants to outrank a specific competitor.

## Workflow

### 1. SERP Analysis

Search target keyword. For top 5 results, note:
- Title structure (how-to? listicle? comparison?)
- Word count
- H2 topics covered
- Unique angle or data
- What they're MISSING (your opportunity)

### 2. Content Gap

Identify gaps:
- Missing subtopics
- Outdated data (>12 months)
- No examples/case studies
- No original research
- Poor mobile formatting

### 3. Structure

Build outline:
1. **Title**: Primary keyword, 50-60 chars, trigger curiosity
2. **Intro**: Hook + problem + solution preview (100-150 words)
3. **H2 sections**: 4-6 covering gaps + competitor topics
4. **Each H2**: 200-400 words, examples, data, practical advice
5. **Conclusion**: Summary + CTA

### 4. On-Page SEO

- Primary keyword in: title, H1, first 100 words, one H2, conclusion
- Secondary keywords in remaining H2s, image alt text
- Meta: 150-155 chars, keyword + benefit
- URL: keyword-only, hyphens, no stop words
- Internal links: 2-3, external: 1-2
- Images: 1 per H2, descriptive alt text

### 5. Write

Prioritize: original data, specific numbers, actionable steps.

## Pitfalls

- Don't regurgitate SERP — Google won't rank a remix
- Keyword stuffing kills readability AND rankings
- <1000 words rarely ranks for competitive terms
- Missing E-E-A-T hurts YMYL topics
