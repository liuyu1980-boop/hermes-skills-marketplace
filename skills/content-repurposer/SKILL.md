# Content Repurposer

Turn one piece of content into platform-optimized versions.

## Trigger

Load when the user provides a URL and wants to repurpose for another platform, or says "turn this into a 小红书 post" / "make this a newsletter" etc.

## Workflow

### 1. Extract Source

Read the URL. Capture: title, main insight, key data/quotes, target audience.

### 2. Identify Targets

Ask which platforms. Support:
- **小红书** — max 1000 chars, emoji-heavy, hook-first, 3-5 bullets
- **Newsletter** — 500-800 words, personal voice, one takeaway
- **Twitter/X** — thread, 5-8 tweets, punchy hooks
- **Blog** — 1500-3000 words, SEO structure, H2/H3

### 3. Adapt Per Platform

**小红书:**
- Hook in first 2 lines (before "..." fold)
- Heavy emoji (1 per 2-3 lines)
- Bullets, not paragraphs
- CTA at end

**Newsletter:**
- Conversational opener
- One core insight, not a summary
- Personal anecdote or opinion

**Twitter/X:**
- Tweet 1 = hook (50% of effort here)
- Middle = evidence
- Last = CTA

**Blog:**
- SEO title (60 chars, primary keyword)
- Meta description (155 chars)
- H2 sections with keyword variants
- Internal links + conclusion with CTA

### 4. Output

Deliver all versions, clearly labeled per platform.

## Pitfalls

- Don't copy-paste with formatting changes — each platform needs a **different angle**
- 小红书 fold: first 2 lines = everything
- Newsletter: if every issue is "here's what I read" = unsubscribes
- Twitter: Tweet 1 IS the battle
